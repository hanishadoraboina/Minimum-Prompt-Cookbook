Minimum Prompt Cookbook

Five prompts that taught me how to communicate effectively with Claude Code. Small prompts. Better results. Bigger lessons.

01 The Clear Goal

Start with intent, not vagueness

prompt.txt 
Create a responsive landing page for my portfolio using HTML and CSS.
What Claude Did

Generated a clean landing page with responsive sections, semantic HTML, and modern styling.

What I Learned

Giving Claude a clear goal produces much better results than asking for "a website."

What Claude Did

Improved only the hero section while preserving the rest of the website.

What I Learned

Providing context helps Claude make focused improvements instead of rewriting everything.

02 Focused Context

Scope the change, protect the rest

prompt.txt 
Using my existing portfolio files, improve the hero section without changing the overall design.

03 Surgical Edits

Say exactly what shouldn't change

prompt.txt 
Update my package.json by adding a "test" script that runs "vitest". Keep all existing scripts unchanged.
What Claude Did

Added only the requested script without changing any other configuration.

What I Learned

Specific prompts create smaller, safer edits that are easier to review.

What Claude Did

Started a fresh session with an empty context window.

What I Learned

Clearing context before changing tasks improves accuracy and avoids confusion.

04 The Reset

Sometimes the best prompt is one command

command 
/clear

05 File-Grounded Prompts

Point at the file, not a description of it

prompt.txt 
@index.html
Improve accessibility while preserving the current design.
What Claude Did

Used the HTML file as context and improved accessibility without changing the appearance.

What I Learned

Giving Claude the exact file it needs leads to much better answers than describing the code manually.
